# プロジェクト固有開発ルール (Dinosaurs Website)

本プロジェクトにおいて、新規の古生物（恐竜等）を追加、あるいはアセットを処理する際は、必ず以下の品質基準および実装手順を厳格に遵守すること。これに違反したアセット配置やコード変更は一切禁止する。

---

## 1. 骨格側面図 (Cyber Asset) の品質基準

サイズ比較シミュレータおよび詳細ページで使用する骨格側面図（`public/images/cyber/*.webp`）は、以下の基準を100%満たす必要がある。

1.  **完全背景透過 (透過WebP / Alpha=0) であること**
    *   `mix-blend-mode: screen` に頼る不透過の黒背景画像は禁止する。
    *   背景の黒およびノイズ（輝度合計値が70以下の薄黒い部分）は、すべて完全に透明（`Alpha = 0`）に加工しなければならない。
    *   出力は必ず **4チャンネル（RGBA）** を持つ **透過WebP（無劣化ロスレスエンコード）** とする。
2.  **1:1 の正方形アセットであり、恐竜の全身が収まっていること**
    *   横長画像を正方形にクロップ（切り抜き）して頭や尾を削ることは厳禁。
    *   必ず左右（頭と尾）を一切削らずに、上下に**透明な余白（Alpha=0）**を追加して1:1の正方形にする（`sharp.extend` 等を使用）。
3.  **AIマークの物理的完全消滅**
    *   画像生成AIによって生成された右下のAIマークは、プログラムによって対象ピクセル領域を物理的に透明（`Alpha = 0`）に上書き消去しなければならない。
4.  **正確な接地とスキャン**
    *   透過加工後の画像に対し、ピクセル走査スクリプトを実行して正確な「恐竜の実体の高さ（`contentHeightPx`）」と「下部余白の高さ（`bottomGap`）」を自動検出すること。手動で推測した値を目分量で入力してはならない。

---

## 2. スケーリング・サイズ調整ルール

恐竜や人間の表示サイズを調整する際は、以下の明確な役割分担に従うこと。

1.  **サイズ比較シミュレータページ内の恐竜サイズの調整**:
    *   シミュレータ（`size-comparison.astro`）は、純粋な1:1の実寸比較を行うため、メタデータの `zoom` 値を意図的に無視する。
    *   したがって、シミュレータ上で恐竜の表示サイズのみを大きく・小さくしたい場合は、**`cyberMetadata.contentHeightPx` を比率計算して調整**すること。
        *   例: 恐竜を 1.15倍 にしたい場合 ➔ `contentHeightPx = (自動スキャン実寸) / 1.15`
        *   例: 恐竜を 0.8倍 にしたい場合 ➔ `contentHeightPx = (自動スキャン実寸) / 0.8`
2.  **詳細（アーカイブ）ページ内の恐竜・人間セットでの調整**:
    *   詳細ページ（`dictionary/[id].astro`）は、人間との比較サイズ全体を大きく・小さく見せるため、メタデータの **`cyberMetadata.zoom`** 値を使用する。
    *   全体のサイズ比較の迫力を増したい（または減らしたい）場合は、`zoom` の値を書き換えること。

---

## 3. 見切れ防止とクランプ仕様

1.  **詳細ページでのクランプ仕様**:
    *   高さ方向: 恐竜がViewBox上端を突き抜けて見切れるのを防ぐため、高さ上限クランプを常に適用する。
    *   幅方向（左右）: `zoom` が明示的に指定されている拡大表示時は、迫力ある表示とはみ出しを許容するため、左右幅のクランプ制限はバイパス（適用除外）とする。`zoom` が未指定の時のみ、安全のための左右クランプを適用する。
