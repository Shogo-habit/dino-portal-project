# Cyber HUD Golden Rules (HUD_GOLDEN_RULES.md)

このドキュメントは、恐竜サイズ比較 HUD における技術的な必須要件を定義します。
**アセットの追加や UI 修正を行う際は、必ず以下のルールを遵守しているか確認してください。**

---

### 1. 完璧な背景背景透過
- **要件**: 黒背景の画像アセット（WebP/PNG）は、背景を完全に除去して表示すること。
- **実装**: `mix-blend-mode: screen !important` を適用する。
- **権限**: `.cyber-hud-specimen` に直接適用し、親コンテナ（`.size-comparison-panel`）では背景を `transparent` に設定して透過を妨げないこと。

### 2. 接地（Baseline Grounding）の徹底
- **要件**: 恐竜と人間（または比較対象）の足元が、正確にネオンのベースライン（下から 100px）上に乗っていること。
- **実装**:
    - `app.js` の `renderSizeComparison` 内で `bottom: 100px` をコンテナに適用。
    - 画像内の実体高度に基づいて算出された `margin-bottom` によるオフセット調整を必須とする。
    - CSS の `object-position: bottom` を併用して位置ずれを防止。

### 3. フレームサイズとスケーリング（Hero-Specimen Scaling）
- **要件**:
    - 比較パネルの高さは **400px 固定** とする。
    - 恐竜標本（ヒーロー）の **「実体（余白を除く）の高さ」** を、枠の高さの **約70%（高さ 280px 相当）** で表示する。
    - 他の比較対象（人間等）は、恐竜の実際の全長・体高に基づく比率に従ってスケーリングする。
- **実装**: 
    - `app.js` の `renderSizeComparison` において、画像全体のサイズではなく `contentHeightPx` を基準にスケールを算出すること。
    - 画像内の下部余白（`bottomGap`）を `margin-bottom` で相殺し、足元をベースラインに密着させること。

### 4. 比較対象の選定とスタイル
- **要件**: 直感的にサイズが伝わる見慣れた対象（人間、SUV、建物等）を使用する。
- **基準**: 人間の高さは **1.7m** 固定。
- **デザイン**: 比較対象はすべて高品質な「サイバー・ブループリント」スタイルを使用すること。

---

### 5. メンテナンスと「不変の定数」ルール
本 HUD の整合性を維持するため、以下の物理定数および開発原則を遵守すること。

- **物理定数**:
    - **パネル全高**: 400px 固定
    - **ベースライン（接地線）**: `bottom: 100px` (グリッド第1主線に一致)
    - **最大標本高度**: 300px (100pxのベースラインから上端までの残り)
- **判定基準（Golden Standard）**:
    - **ティラノサウルス、トリケラトプス、ステゴサウルス**の各ページにおける計算・表示ロジックを「正典」とする。
- **変更不可の原則**:
    - 新しい恐竜の追加や個別の修正において、**共通のベースライン（100px）やパネル全高（400px）を変更してはならない。** 
    - 特定の恐竜が枠からはみ出す場合は、ベースラインを動かすのではなく、その恐竜の `pxPerMeter`（動的スケール）またはメタデータの数値を調整して解決すること。
    - **「完成済み」の他のページを破壊してはならない。**
